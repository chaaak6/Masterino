# MasterLion Handoff

## 1. 项目目标

MasterLion 是从 LobeHub 迁移来的小宗狮 AI 应用，目标是形成可本地部署、可对接 Aihub 的企业 AI 工作台。

核心目标：

- 小宗狮应用不直接连接 Aihub 数据库，通过独立 `aihub-db-bridge` 服务读取 Aihub 用户、托管 token、模型、额度和日志数据。
- 用户 `10193226` 可作为真实验收账号完成登录、模型选择、Aihub 余额查看、真实 AI 对话和文件上传入口测试；密码不写入仓库文档。
- Aihub 模型按用户组和 token model limits 过滤；默认模型为当前用户可用模型列表第一个。
- 余额展示为人民币金额和 token/request 行式数据，不直接把 raw quota 当余额。
- 文件上传走本地 RustFS/S3；浏览器上传优先使用 MasterLion 同源上传代理，避免 RustFS CORS 阻塞。
- 离线/本地部署时隐藏或降级依赖 LobeHub 官方 market、云端沙箱等远程能力。

## 3. 已完成修改

已完成并发版验证：

- 新增/完善 Aihub DB Bridge：
  - 独立 `apps/aihub-db-bridge` 服务镜像。
  - 小宗狮主应用通过 bridge 查询 Aihub 绑定、用户组、托管 token、模型、余额和 usage logs。
  - 不把 Aihub 数据库连接混进主应用。
- Aihub 模型过滤：
  - 读取 Aihub 用户组、abilities 和 token model limits。
  - 模型列表 = group enabled abilities 与 token model limits 的交集。
  - token 未启用 model limits 时，只按用户组 abilities。
  - default 用户组不会看到 VIP-only 模型。
  - `ai_models` 只保存过滤后的可用模型。
  - `ai_providers.check_model` 写入过滤后的第一个可用模型。
- Aihub 余额展示：
  - 页面显示人民币金额、已用金额、请求数、Prompt Token、Completion Token、Total Token。
  - 金额使用 `¥x.xx`，金额区域不换行。
  - 使用 Aihub `/api/status` 的 `quota_per_unit` 和 `usd_exchange_rate`；失败时 fallback 到环境变量默认值。
- 文件上传：
  - 增加同源上传代理 `/api/upload/s3-proxy`。
  - 浏览器不再直接使用 `rustfs:9000` 内网地址。
  - 上传代理被 middleware 放行，并保留签名校验。
  - `OPTIONS /api/upload/s3-proxy` 返回 200 和 CORS 响应头。
  - 未签名 `PUT` 返回 403。
- RustFS 初始化：
  - RustFS bucket CORS 设置改为 best-effort。
  - RustFS CORS API 仍会返回 `decoding xml: EOF`，但不阻塞，因为浏览器上传走同源代理。
- 文件分析降级：
  - 上传成功和 embedding/解析失败解耦。
  - embedding 不可用时不应阻断当前会话附件使用。
- 云端沙箱：
  - 未接入完整自托管 onlyboxes。
  - 未配置时隐藏或降级官方沙箱入口。
  - 短期替代方案是本地服务端落盘/S3 上传并返回 `/f/:id` 下载链接。
- 官方 market 推荐：
  - 新增 `MASTERLION_ENABLE_MARKET_RECOMMENDATIONS=1` 开关。
  - 默认关闭官方推荐请求，避免离线部署时报 `seedKey required`。

  - `Dockerfile` 增加 `CI=true`，修复 pnpm 非 TTY build 问题。

## 5. 不能动的边界

- 不要把真实密钥、真实 DB 连接串、`.env`、`.env.desktop`、`.codex/`、`.next/`、`node_modules/`、本地日志或浏览器缓存提交到 GitHub。
- 不要让小宗狮主应用直接连接 Aihub 数据库；Aihub DB 读取必须通过独立 bridge/service 镜像。
- 不要把 Aihub 用户 token 通过普通管理员 API 暴露给前端；获取具体 token 仍需要 bridge/DB 受控读取。
- 不要默认展示用户组无权限模型；模型必须按 group abilities 和 token model limits 过滤。
- 不要把 raw quota 直接当余额展示；余额必须是人民币金额，并同时展示 token/request 数据。
- 不要在开发期每次修改都重建镜像；只有确认发版才 build image。
- 不要依赖外网官方 market/skill/MCP/sandbox 才能进入应用；离线时必须隐藏、缓存或降级。
- 不要回退当前未提交改动，除非用户明确要求。

## 6. 已经否掉的方案

- 否掉“主应用直接连 Aihub DB”：
  - 会把 DB 连接混进小宗狮应用，不符合隔离要求。
- 否掉“只用 Aihub 管理员 API 获取具体 token”：
  - 已确认除 DB/bridge 外，管理员 API 不稳定或不适合直接读取具体托管 token 明文。
- 否掉“浏览器直接 PUT 到 RustFS 内网 endpoint”：
  - `rustfs:9000` 浏览器不可达，且 RustFS CORS API 不可靠。
  - 已改为同源 `/api/upload/s3-proxy`。
- 否掉“短期完整接入云端沙箱 onlyboxes”：
  - 成本高，需要部署 onlyboxes、鉴权签名、资源隔离、文件导出通道和网络策略。
  - 当前先隐藏/降级，文件下载走本地 `/f/:id`。
- 否掉“热更新替代 Docker 发版”：
  - 用户反馈热更新卡顿/崩溃，当前回到原 Docker 模式。
- 否掉“默认固定 glm5.1”：
  - 最新要求是默认使用当前用户过滤后模型列表的第一个。
